# 阶段 0002：`llstk init`

## 目标

实现第一个真实写入命令，初始化一个可读、可手工检查、可被后续命令继续使用的 `.locallab` 工作区。

## 前置条件

- 阶段 0000 和 0001 已完成。
- `RootManifest` 和 `WorkspaceLayout` 已可用。
- CLI 已支持全局 `--root <path>` 或等价工作区路径配置。

## 命令规格

```bash
llstk init [--name <name>] [--force]
```

参数：

- `--name <name>`：根清单中的 stack 名称，默认 `default`。
- `--force`：允许覆盖由 LocalLabStack 生成的文件；第一版可以只支持覆盖根清单、根 compose 和默认 Caddyfile。

如果阶段 0001 已实现全局 `--root`，则以下命令也必须可用：

```bash
llstk --root .my-lab init
```

## 创建内容

默认运行 `llstk init` 应创建：

```text
.locallab/
  docker-compose.yml
  llstk.yml
  README.md
  gateway/
    Caddyfile
    data/
    config/
  certs/
    ca/
    issued/
  templates/
```

## 文件内容要求

根清单 `.locallab/llstk.yml` 必须符合阶段 0001 的 `RootManifest`。

根 compose `.locallab/docker-compose.yml` 必须包含：

- `gateway` service。
- `caddy:2-alpine` image。
- `locallabstack-gateway` container name。
- `80:80` 和 `443:443` ports。
- Caddyfile、data、config、certs volumes。
- `locallabstack-global` network。

默认 `.locallab/gateway/Caddyfile` 必须是可被 Caddy 解析的占位配置，至少包含：

```caddyfile
# Generated by LocalLabStack. Manual edits may be overwritten.

{
  auto_https off
}

*.locallab {
  tls /certs/issued/locallab.pem /certs/issued/locallab-key.pem
  respond "Unknown LocalLabStack app" 404
}
```

README 必须说明：

- 该目录由 LocalLabStack 管理。
- 文件可以人工检查。
- 私钥不应提交到版本库。
- 下一步可以运行 `llstk app create ...`。

## 写入策略

- 目录不存在时创建目录。
- 目录已存在时继续创建缺失目录。
- 文件不存在时写入默认内容。
- 文件已存在且没有 `--force` 时不得覆盖，应输出 `exists, skipped`。
- 文件已存在且传入 `--force` 时可以覆盖，但只覆盖本阶段明确生成的文件。
- 不得删除用户已有文件。

## 输出要求

成功时输出每个创建或跳过的主要路径，例如：

```text
created .locallab/llstk.yml
created .locallab/docker-compose.yml
created .locallab/gateway/Caddyfile
created .locallab/certs/issued/
```

重复运行时输出：

```text
exists, skipped .locallab/llstk.yml
```

## 具体任务

- [ ] 在 `commands::init` 中实现 `llstk init`。
- [ ] 使用阶段 0001 的 workspace 路径模型，不手写散落路径。
- [ ] 使用 `RootManifest` 默认值生成根清单。
- [ ] 生成根级 Docker Compose 文件。
- [ ] 生成默认 Caddyfile。
- [ ] 创建 gateway、certs、templates 等目录。
- [ ] 实现 `--name`。
- [ ] 实现非破坏性重复运行。
- [ ] 为临时目录中的 init 行为添加集成测试。

## 错误场景

- 目标路径存在但不是目录时，应失败并提示路径。
- 文件写入失败时，应失败并提示路径和底层错误。
- `--name` 为空时应失败。
- `.locallab/llstk.yml` 存在但不是合法文件时，`init` 不应尝试修复，除非用户传入 `--force`。

## 测试要求

至少覆盖：

- 空临时目录运行 `llstk init` 后结构完整。
- 重复运行 `llstk init` 不覆盖用户修改过的文件。
- `llstk init --force` 覆盖允许覆盖的生成文件。
- `llstk --root custom init` 在 custom 目录下创建结构。
- 根清单可以被 `RootManifest` 读回。
- 生成的 compose 文本包含 gateway service 和全局网络。
- 生成的 Caddyfile 包含 wildcard fallback。

## 验收标准

本阶段完成时，以下命令必须通过：

```bash
cargo fmt --check
cargo test
cargo run --bin llstk -- init
cargo run --bin llstk -- init
```

第二次 `init` 必须安全跳过已有文件，不破坏第一次生成的内容。

## 本阶段不做

- 不创建任何应用目录。
- 不渲染真实应用路由。
- 不生成证书。
- 不启动 Docker Compose。
- 不写入 `/etc/hosts`。
